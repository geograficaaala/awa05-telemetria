Sistema de Telemetría Ambiental

Monitoreo continuo de nivel de captación de agua y variables microclimáticas mediante Raspberry Pi 4B, sensor ultrasónico JSN-SR04T y estación meteorológica Ambient Weather WS-2000. Datos procesados localmente y publicados en tiempo real.

Arquitectura del sistema

Flujo de adquisición y publicación

El nodo opera sobre Raspberry Pi 4B ejecutando un servicio systemd persistente que coordina la adquisición de nivel por ultrasonido, la ingesta meteorológica por HTTP y la sincronización periódica con GitHub Pages.

Etapa 01
Adquisición de nivel
Lectura por eco ultrasónico JSN-SR04T con promediado de mediana en 5 muestras. Conversión geométrica a volumen en litros.
Etapa 02
Ingreso meteorológico
Servidor Flask en puerto 7777 recibe datos de la WS-2000 vía protocolo Wunderground. Persistencia en CSV local.
Etapa 03
Procesamiento
Pipeline Python genera dashboard_data.json con series temporales, KPIs actuales y diagnóstico del hardware.
Etapa 04
Publicación web
Sincronización con GitHub vía PyGithub API. Tablero estático desplegado en GitHub Pages con actualización automática.

Variables monitoreadas

Nivel de agua
JSN-SR04T
Distancia convertida a volumen aplicando geometría cilíndrica del contenedor.
Temperatura exterior
WS-2000
Serie térmica ambiental. Referencia para evaluación de condiciones de condensación.
Humedad relativa
WS-2000
Indicador central para el potencial de captación de agua atmosférica.
Punto de rocío
WS-2000
Temperatura de condensación. Variable crítica para interpretar la captación.
Presión barométrica
WS-2000
Seguimiento de variaciones para interpretación de patrones atmosféricos locales.
Viento
WS-2000
Velocidad y dirección independientes para análisis de intercambio de masa.
Precipitación horaria
WS-2000
Registro mm/h para distinguir captación por condensación de lluvia directa.
Radiación solar y UV
WS-2000
Carga radiativa instantánea para contextualizar el desempeño térmico del sistema.
Infraestructura

Componentes del nodo

Raspberry Pi 4B
Unidad de procesamiento central. Ejecuta scheduler, servidor Flask y cliente GitHub bajo servicio systemd.
Debian Trixie · 64-bit
JSN-SR04T
Transductor ultrasónico de nivel. Pines BCM 17 (TRIG) y 18 (ECHO) con divisor de voltaje 5V a 3.3V.
GPIO BCM · Mediana 5 muestras
WS-2000
Estación meteorológica Ambient Weather. Transmite por WiFi al endpoint Flask en puerto 7777 vía Wunderground.
Puerto 7777 · Intervalo 60 s
GitHub Pages
Publicación del sitio estático desde la raíz de la rama main. Actualización automática por push desde la Pi.
