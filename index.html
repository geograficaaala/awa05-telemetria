Flujo del sistema

El nodo opera sobre Raspberry Pi, captura nivel de agua por sensor ultrasónico, recibe datos meteorológicos por HTTP desde la WS-2000, consolida archivos en formato CSV y expone la información al tablero web publicado en GitHub Pages.

Etapa 01
Adquisición de nivel
Lectura de distancia mediante JSN-SR04T y conversión geométrica a volumen de agua en litros.
Etapa 02
Ingreso meteorológico
Recepción de temperatura, humedad, punto de rocío, presión, viento, lluvia, radiación solar y UV desde la estación WS-2000.
Etapa 03
Persistencia local
Almacenamiento estructurado en data/raw/nivel_raw.csv y data/raw/clima_raw.csv.
Etapa 04
Publicación web
Generación del dataset del tablero y despliegue estático mediante GitHub Pages con acceso desde la raíz del repositorio.

Variables monitoreadas

Cobertura del modelo meteorológico

Nivel de agua
Sensor ultrasónico
Conversión de distancia a volumen a partir de la geometría del contenedor y el rango operativo del sensor.
Temperatura exterior
WS-2000
Serie térmica del ambiente exterior utilizada para evaluación de condiciones de condensación.
Humedad relativa
WS-2000
Indicador central para el potencial de captura de agua atmosférica y comparación con producción registrada.
Punto de rocío
WS-2000
Variable derivada clave para interpretar el umbral de condensación y su relación con el nivel de agua acumulado.
Presión barométrica
WS-2000
Seguimiento de variaciones barométricas como apoyo a la interpretación de cambios en las condiciones atmosféricas locales.
Viento
WS-2000
Visualización independiente para velocidad y dirección, permitiendo análisis del intercambio de masa y la ventilación del sistema.
Lluvia por hora
WS-2000
Registro de precipitación horaria para distinguir captación por condensación frente a episodios de lluvia local.
Radiación solar y UV
WS-2000
Seguimiento de carga radiativa mediante radiación solar y nivel UV para contextualizar el desempeño térmico del entorno.
