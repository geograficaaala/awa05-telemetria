SISTEMA AWA05

Producción hídrica, clima y estado operativo en una sola plataforma

Este sitio presenta la operación del nodo AWA05 instalado en campo. Una Raspberry Pi 4B registra el nivel de captación, las variables microclimáticas y la salud del hardware; después procesa y publica los datos de forma automática cada quince minutos. El resultado es una vista pública y actualizada del desempeño de la máquina cosechadora de agua.

Módulos de monitoreo

El sistema está organizado en ocho módulos que resumen la producción, el clima, la operación del nodo y el desempeño económico de AWA05.

01
⬡
Resumen ejecutivo
Seis indicadores en tiempo real: nivel de cisterna, garrafones producidos, humedad, temperatura, punto de rocío y viento.
Tiempo real
02
◈
Salud del nodo Pi
Temperatura de CPU, voltaje, uso de RAM y disco, uptime, señal WiFi e IP Tailscale. El sistema emite alerta de apagado a 75 °C para proteger el hardware.
Hardware
03
◉
Producción hídrica
Gráfica del nivel de agua en la cisterna a lo largo del tiempo. Los puntos verdes marcan cada momento en que se completó un garrafón de 20 litros.
Sensor JSN-SR04T
04
◑
Análisis diurno / nocturno
La captación de niebla en el lago de Atitlán es predominantemente nocturna. Este módulo descompone la producción por franja horaria y por día de experimento.
Análisis temporal
05
◎
Análisis empresarial
Calcula el costo de producir un litro y un garrafón, además del ahorro frente al precio de mercado, usando la tarifa eléctrica local y escenarios de consumo.
Costos · Q/L · Q/garrafón

WS-2000
07

Velocidad y dirección del viento, precipitación horaria y radiación solar. Estas variables ayudan a explicar las condiciones de captación de niebla.

Arquitectura del sistema
Del sensor al tablero

Cuatro etapas automáticas conectan la medición física con la visualización pública de los datos.

01
Adquisición
El sensor JSN-SR04T mide el nivel cada 15 minutos. La WS-2000 envía clima cada 60 segundos al servidor Flask interno.
02
Procesamiento
Python filtra outliers, convierte unidades, calcula KPIs y genera el archivo JSON del tablero.
03
Publicación
PyGithub sube el JSON al repositorio y GitHub Pages despliega automáticamente los datos actualizados.
04
Visualización
El tablero carga el JSON al abrirse y se refresca cada 60 segundos mediante HTML estático.
Cobertura de datos
Variables monitoreadas

Nueve series climáticas y la medición de nivel permiten estudiar cuándo y cuánto agua produce la máquina.

Nivel de agua
JSN-SR04T
Distancia ultrasónica convertida a volumen cilíndrico. Radio 12.9 cm · Altura útil 38.3 cm · Máx 20 L.
Humedad relativa
WS-2000
Variable clave para la formación de niebla y la captación. El análisis evalúa su relación con la producción.
Temperatura exterior
WS-2000
Serie térmica ambiental que influye en la evaporación y la condensación nocturna. Convertida de °F a °C.
Punto de rocío
WS-2000
Temperatura de condensación. Cuando se acerca a la temperatura ambiental, la humedad relativa tiende a aumentar.
Presión barométrica
WS-2000
Las variaciones de presión ayudan a identificar cambios de masas de aire y condiciones meteorológicas.
Velocidad del viento
WS-2000
Variable física relevante para transportar las gotas de niebla hacia la superficie colectora.
Dirección del viento
WS-2000
Analizada por sectores y puntos cardinales para estimar la exposición de la máquina al flujo de niebla.
Precipitación horaria
WS-2000
Registro en mm/h para diferenciar los aportes de lluvia directa y los eventos de condensación.
Radiación solar y UV
WS-2000
Variables asociadas al ciclo diurno y a las condiciones que favorecen o reducen la captación.
Infraestructura de campo
Componentes del nodo

Hardware de bajo consumo preparado para operar de forma continua y publicar telemetría desde el sitio de monitoreo.

Raspberry Pi 4B
Unidad central de procesamiento. Ejecuta el scheduler, el servidor Flask y el cliente GitHub bajo un servicio systemd persistente. Incluye supervisión térmica y apagado preventivo.
Debian Trixie · 64-bit · ARM
JSN-SR04T
Transductor ultrasónico para medición de nivel. Utiliza la mediana de 11 muestras para reducir ruido y está conectado mediante GPIO.
GPIO BCM · Mediana 11 muestras
WS-2000
Estación meteorológica que transmite por WiFi al endpoint Flask interno mediante un protocolo compatible con Wunderground.
Puerto 7777 · Cada 60 s
Tailscale VPN
Acceso remoto seguro al nodo desde redes autorizadas. Permite administración SSH sin exponer directamente el puerto a Internet.
WireGuard · Mesh VPN
GitHub Pages
Publicación del tablero estático desde la rama principal. Los datos procesados se actualizan automáticamente desde la Raspberry Pi.
Rama main · Raíz / · Estático
